# ADR-0002: ルール中心のスキーマ

* ステータス: 承認済み

## 決定

FS Schemaの表現方式として、ツリー中心ではなくルール中心を採用します。

ルールは`rules`配下に列挙し、各ルールで対象と制約を表現します。

## 理由

本ライブラリで扱いたい要件は、木構造の定義よりも制約の列挙に寄っています。

* このファイルが必要
* AがあればBも必要
* 特定のディレクトリ配下は特定の条件を満たす
* ファイルサイズや名前などの条件を満たす

これらはツリー定義として表現するよりも、ルールの並びとして表現する方が自然です。

また、対象の選択と制約を分けて表現しやすくなるため、今後のルール追加や拡張にも対応しやすくなります。
