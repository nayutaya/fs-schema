# ADR-0003: ルールのidを省略可能にする

* ステータス: 承認済み

## 決定

各ルールの`id`プロパティを必須ではなく省略可能にします。

`id`が指定されている場合はそれをルール識別子として扱い、指定されていない場合は実装側で`rule{i}`形式の内部ルール識別子を生成します。

ここで`i`はルール配列中の0始まりのインデックスとします。

## 理由

単純な使用例や小規模なスキーマでは、各ルールに一意な`id`を付けることが記述上の負担になります。
特に、単一の`require`や`forbid`だけを記述するようなケースでは、`id`がなくてもルールの意図は十分に読み取れます。

また、type別の使用例を増やしていく場合、`id`の記述を必須にすると最小構成の例でも補助的な記述が増え、スキーマの要点が見えにくくなります。

一方で、`id`は診断結果やデバッグ時の参照に有用であるため、プロパティ自体は残し、必要な場面では引き続き指定できるようにするのが妥当です。

## 検討事項

内部ルール識別子は入力順に依存するため、ルールの並び替えを行うと診断結果上の識別子も変わります。

そのため、安定した識別子が必要なケースでは、引き続き利用者が明示的に`id`を指定する運用が適しています。
