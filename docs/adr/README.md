# Architecture Decision Records

このディレクトリには、FS Schemaの設計上の重要な判断をArchitecture Decision Record（ADR）として記録します。

## 書き方

ADRは、あとから設計判断の背景を追えるように簡潔に書いてください。

基本的には次の構成にします。

```md
# ADR-0001: タイトル

* ステータス: 承認済み

## 決定

決定した内容を書きます。

## 理由

なぜその決定にしたのかを書きます。

## 検討したが採用しなかった案

必要に応じて、代替案と採用しなかった理由を書きます。

## 影響

必要に応じて、実装・利用者・将来の設計への影響を書きます。
```

すべての見出しを必ず使う必要はありません。
ただし、少なくとも「決定」と「理由」は書いてください。

## ファイル名

ファイル名は次の形式にしてください。

```txt
NNNN_short_title.md
```

* `NNNN`は4桁の連番にしてください
* 既存ADRの最大番号の次の番号を使ってください
* `short_title`は英小文字のsnake_caseにしてください
* ファイル名は変更内容が分かる短い名前にしてください
